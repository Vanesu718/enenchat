## 🚀 使用建议

### 方案1：继续使用单文件版本（推荐）
**优点：**
- 无需服务器，直接双击打开
- 所有功能集成在一个文件中
- 易于分享和部署
- 已支持docx文档上传

**使用方法：**
直接使用 `index.html`，所有功能都已完整实现。

### 方案2：使用多文件结构（适合开发）
**优点：**
- 代码结构清晰
- 易于维护和扩展
- 团队协作友好

**注意事项：**
- 需要通过HTTP服务器运行（如Live Server）
- 不能直接双击打开（会有CORS限制）

- **前端框架：** 纯JavaScript（无框架）
- **样式：** CSS3（支持backdrop-filter毛玻璃效果）
- **存储：** localStorage
- **文档解析：** mammoth.js（用于.docx文件）
- **AI集成：** OpenAI API兼容接口

## 🐛 已知问题

1. **localStorage容量限制**
   - 浏览器通常限制5-10MB
   - 大量图片可能导致存储失败
   - 建议使用链接上传或压缩图片

2. **docx支持限制**
   - 仅支持.docx格式（Word 2007+）
   - 不支持.doc（旧版Word）和.wps格式
   - 复杂格式可能丢失（仅提取纯文本）

## 📞 技术支持

如需进一步优化或遇到问题，可以：
1. 检查浏览器控制台的错误信息
2. 清空localStorage重新开始
3. 使用数据备份功能保存重要数据
