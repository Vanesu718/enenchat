# 为什么别人的小手机能存很多图片？

## 真相揭秘

你看到的那些"能存很多图片"的小手机应用，其实用的是以下几种方法：

### 1. 使用后端服务器存储 💾
- 图片上传到服务器，localStorage 只存图片链接
- 需要租用服务器（每月几十到几百元）
- 例如：阿里云OSS、腾讯云COS

### 2. 使用 IndexedDB 数据库 📊
- 浏览器的另一种存储方式，容量可达 50MB-1GB
- 但需要完全重写代码架构
- 技术难度较高

### 3. 使用图床服务 🖼️
- 图片自动上传到免费图床
- localStorage 只存链接
- 常见图床：imgur、sm.ms、imgbb

### 4. 做成桌面应用 💻
- 使用 Electron 等技术打包成桌面软件
- 可以直接访问电脑硬盘，没有存储限制
- 但不再是网页，需要安装

---

## 关于"发送到桌面"的误解

你提到的"发送到桌面使用"，这个功能叫 **PWA（渐进式Web应用）**：

❌ **错误理解：** 发送到桌面后，数据就存在手机/电脑上了
✅ **实际情况：** 数据仍然存在浏览器的 localStorage 里，限制不变

**PWA 只是创建了一个快捷方式，本质还是网页！**

## 关于图片压缩的担心

你担心压缩后图片不好看？其实：

### 智能压缩不会影响视觉效果 ✨

现代压缩算法非常智能：
- 压缩到 **原大小的 30%**，肉眼几乎看不出区别
- 只去除冗余数据，不损失画质
- 分辨率保持不变

### 实际对比

| 压缩前 | 压缩后 | 视觉效果 |
|--------|--------|----------|
| 2MB | 600KB | 几乎一样 |
| 1MB | 300KB | 几乎一样 |
| 500KB | 150KB | 几乎一样 |

**推荐压缩工具：**
- tinypng.com（最好用，压缩率高）
- squoosh.app（Google 出品）
- compressor.io
